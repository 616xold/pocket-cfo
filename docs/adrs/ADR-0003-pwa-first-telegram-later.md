# ADR-0003: PWA first, messaging bridges later

- Status: Accepted
- Date: 2026-03-08

## Context

Pocket CTO needs a pocket surface, but Telegram or WhatsApp would constrain the interface early and distract from the core proof bundle and replay experience.

## Decision

Ship a mobile-friendly web interface first and treat messaging bridges as later distribution channels.

## Consequences

Good:

- richer evidence cards
- fewer channel constraints
- faster iteration on approvals and replay UI

Trade-offs:

- less instant novelty than a messaging bot
